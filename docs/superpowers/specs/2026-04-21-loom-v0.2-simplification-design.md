# Loom v0.2 — Simplification Refactor Design

**Date:** 2026-04-21
**Status:** Approved for implementation planning
**Target branch:** `refactor/v0.2-sqlite-networkx`

---

## Purpose

Strip Loom from a FalkorDB + embedding + linker + drift + ticket + watcher stack (~12,700 LOC, 87 source files, Docker-required) down to its defensible core: **call graph + blast radius + MCP server, ship-anywhere, zero infrastructure**.

End state: `pip install loom-tool` works on any machine, no Docker, no model download, no Redis. Single SQLite file at `~/.loom/loom.db`. MCP server starts immediately.

---

## Reference projects studied

- **`safishamsi/graphify`** — NetworkX + tree-sitter, in-memory, 71.5× claimed token reduction. Source: `LanguageConfig` generic walker, three-tier confidence, SHA-256 cache, 7 granular MCP tools.
- **`tirth8205/code-review-graph`** — SQLite + NetworkX + FastMCP, SHA-256 incremental, 28 MCP tools, auto-install for 11 AI tools, 8.2× token reduction. Source: SQLite recursive CTE blast radius, `tree-sitter-language-pack`, `file_hash`-on-node, `BEGIN IMMEDIATE` atomic per-file replace, ProcessPoolExecutor parse.

Loom keeps what it does better than both: Pydantic-strict models, layered architecture, async-native, typed ID convention, optional doc ingestion.

---

## Goals

1. `pip install loom-tool` — zero external services, zero model downloads.
2. `loom analyze .` on a ~10k-LOC repo in under 30s cold.
3. Blast radius queries < 200ms on graphs up to 50k nodes (SQLite recursive CTE).
4. MCP server starts in under 1s, exposes 10 focused tools.
5. `loom install` configures Claude Code / Cursor / Windsurf / Zed / Codex automatically.

## Non-goals (v0.2)

- Semantic search via embeddings — use FTS5 instead.
- Doc ↔ code linking (`LOOM_IMPLEMENTS` edges) — removed; add back as optional plugin in v0.3.
- Drift detection — removed.
- Jira / GitHub / Linear ticket integration — removed.
- File watcher — replaced by git post-commit hook.
- Unifying 8 parsers into `LanguageConfig` generic walker — deferred to v0.3.

## Data model

### SQLite schema (`src/loom/core/db.py`)

```sql
PRAGMA journal_mode=WAL;
PRAGMA synchronous=NORMAL;
PRAGMA foreign_keys=ON;

CREATE TABLE IF NOT EXISTS nodes (
    id              TEXT PRIMARY KEY,
    kind            TEXT NOT NULL,
    source          TEXT NOT NULL,
    name            TEXT NOT NULL,
    path            TEXT NOT NULL,
    start_line      INTEGER,
    end_line        INTEGER,
    language        TEXT,
    content_hash    TEXT,
    file_hash       TEXT,
    summary         TEXT,
    is_dead_code    INTEGER NOT NULL DEFAULT 0,
    community_id    TEXT,
    metadata        TEXT NOT NULL DEFAULT '{}',
    updated_at      INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_nodes_name ON nodes(name);
CREATE INDEX IF NOT EXISTS idx_nodes_path ON nodes(path);
CREATE INDEX IF NOT EXISTS idx_nodes_kind ON nodes(kind);
CREATE INDEX IF NOT EXISTS idx_nodes_lang ON nodes(language);

CREATE TABLE IF NOT EXISTS edges (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    from_id          TEXT NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
    to_id            TEXT NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
    kind             TEXT NOT NULL,
    confidence       REAL NOT NULL DEFAULT 1.0,
    confidence_tier  TEXT NOT NULL DEFAULT 'extracted',
    metadata         TEXT NOT NULL DEFAULT '{}',
    UNIQUE(from_id, to_id, kind)
);
CREATE INDEX IF NOT EXISTS idx_edges_from      ON edges(from_id);
CREATE INDEX IF NOT EXISTS idx_edges_to        ON edges(to_id);
CREATE INDEX IF NOT EXISTS idx_edges_kind      ON edges(kind);
CREATE INDEX IF NOT EXISTS idx_edges_to_kind   ON edges(to_id, kind);
CREATE INDEX IF NOT EXISTS idx_edges_from_kind ON edges(from_id, kind);

CREATE VIRTUAL TABLE IF NOT EXISTS nodes_fts USING fts5(
    id UNINDEXED, name, summary, path,
    content='nodes', content_rowid='rowid',
    tokenize='porter unicode61'
);
CREATE TRIGGER IF NOT EXISTS nodes_ai AFTER INSERT ON nodes BEGIN
    INSERT INTO nodes_fts(rowid, id, name, summary, path)
    VALUES (new.rowid, new.id, new.name, new.summary, new.path);
END;
CREATE TRIGGER IF NOT EXISTS nodes_ad AFTER DELETE ON nodes BEGIN
    INSERT INTO nodes_fts(nodes_fts, rowid, id, name, summary, path)
    VALUES ('delete', old.rowid, old.id, old.name, old.summary, old.path);
END;
CREATE TRIGGER IF NOT EXISTS nodes_au AFTER UPDATE ON nodes BEGIN
    INSERT INTO nodes_fts(nodes_fts, rowid, id, name, summary, path)
    VALUES ('delete', old.rowid, old.id, old.name, old.summary, old.path);
    INSERT INTO nodes_fts(rowid, id, name, summary, path)
    VALUES (new.rowid, new.id, new.name, new.summary, new.path);
END;

CREATE TABLE IF NOT EXISTS schema_meta (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

### Node model (`core/node.py`)

**Kept NodeKind values:** `FUNCTION`, `METHOD`, `CLASS`, `MODULE`, `FILE`, `INTERFACE`, `ENUM`, `TYPE`, `COMMUNITY`, `DOCUMENT`, `SECTION`, `CHAPTER`, `SUBSECTION`, `PARAGRAPH`.

**Dropped:** `TICKET`.

**NodeSource:** `CODE`, `DOC` (drop `TICKET`).

**Dropped fields:** `embedding`, `status`, `priority`, `assignee`, `url`, `external_id`, `page_start`, `page_end`.

**Added field:** `file_hash: str | None` — per-file SHA-256 stored on every node row. Denormalized: all nodes for a given path share the same `file_hash` value. Lets `sync` skip unchanged files without a separate state table.

**ID convention unchanged:**
- Code: `{kind}:{path}:{symbol}` — e.g. `function:src/auth.py:validate_user`
- Doc: `doc:{path}:{section_ref}` — e.g. `doc:specs/auth.pdf:chapter_3.2.4`

Factory methods: `Node.make_code_id()`, `Node.make_doc_id()`. `Node.make_ticket_id()` removed.

### Edge model (`core/edge.py`)

```python
class EdgeType(StrEnum):
    CALLS = "calls"
    CONTAINS = "contains"
    EXTENDS = "extends"
    IMPORTS = "imports"
    COUPLED_WITH = "coupled_with"
    MEMBER_OF = "member_of"
    CHILD_OF = "child_of"
    REFERENCES = "references"

class ConfidenceTier(StrEnum):
    EXTRACTED = "extracted"   # direct AST evidence (definitive)
    INFERRED  = "inferred"    # cross-file name resolution matched
    AMBIGUOUS = "ambiguous"   # multiple matches, best-guess
```

**Dropped EdgeType values:** `IMPLEMENTS`, `USES_TYPE`, `STEP_IN`, `DYNAMIC_CALL`, `REFLECTS_CALL`, `DYNAMIC_IMPORT`, `UNRESOLVED_CALL`, `LOOM_IMPLEMENTS`, `LOOM_VIOLATES`, `REALIZES`, `CLOSES`, `VERIFIED_BY`, `DEPENDS_ON`.

**Dropped Edge fields:** `origin` (EdgeOrigin), `link_method`, `link_reason`.

**Added Edge field:** `confidence_tier: ConfidenceTier = ConfidenceTier.EXTRACTED` — Pydantic default mirrors SQL default.

Numeric `confidence: float` kept for ranking precision. Tier is the discrete, honest label; float is a refinement.

**Community storage note:** `community_id` on `nodes` is a scalar tag pointing to a `COMMUNITY`-kind node row (materialized community record). `get_community(community_id)` returns all nodes with matching `community_id`. No separate `communities` table.

**FTS5 fallback note:** If FTS5 is unavailable (rare; some Python sqlite3 builds strip it), DDL skips `nodes_fts` table + triggers, and `search()` falls back to `WHERE name LIKE ?`. Detect at schema init by probing `CREATE VIRTUAL TABLE` in a savepoint.

## Core API

```python
# src/loom/core/graph.py
class LoomGraph:
    def __init__(self, db_path: Path | None = None) -> None: ...

    async def bulk_upsert_nodes(self, nodes: list[Node]) -> None: ...
    async def bulk_upsert_edges(self, edges: list[Edge]) -> None: ...

    async def replace_file(
        self, path: str, nodes: list[Node], edges: list[Edge]
    ) -> None:
        """Atomic per-file replace. BEGIN IMMEDIATE → DELETE WHERE path=? → INSERT."""

    async def get_node(self, node_id: str) -> Node | None: ...
    async def get_nodes_by_name(self, name: str, limit: int = 10) -> list[Node]: ...
    async def get_content_hashes(self) -> dict[str, str]: ...
    async def get_file_hash(self, path: str) -> str | None: ...

    async def blast_radius(self, node_id: str, depth: int = 3) -> list[Node]:
        """Recursive CTE over edges WHERE kind='calls'. Returns nodes with _depth."""

    async def neighbors(
        self, node_id: str, depth: int = 1,
        edge_types: list[EdgeType] | None = None,
    ) -> list[Node]: ...

    async def search(self, query: str, limit: int = 10) -> list[Node]:
        """FTS5 over (name, summary, path)."""

    async def shortest_path(self, from_id: str, to_id: str) -> list[Node] | None: ...
    async def community_members(self, community_id: str) -> list[Node]: ...
    async def god_nodes(self, limit: int = 20) -> list[Node]:
        """Highest in-degree nodes on CALLS subgraph."""
    async def stats(self) -> dict[str, int | dict[str, int]]: ...
```

### Blast radius: SQLite recursive CTE

```python
async def blast_radius(self, node_id: str, depth: int = 3) -> list[Node]:
    def _run() -> list[Node]:
        with self._lock:
            conn = self._connect()
            rows = conn.execute(
                """
                WITH RECURSIVE impacted(id, depth) AS (
                    SELECT ?, 0
                    UNION
                    SELECT e.from_id, i.depth + 1
                      FROM edges e
                      JOIN impacted i ON e.to_id = i.id
                     WHERE e.kind = 'calls' AND i.depth < ?
                )
                SELECT n.*, i.depth AS _depth
                  FROM impacted i JOIN nodes n ON n.id = i.id
                 WHERE i.id != ?
                 ORDER BY i.depth, n.name
                """,
                (node_id, depth, node_id),
            ).fetchall()
            return [_row_to_node(r) for r in rows]
    return await asyncio.to_thread(_run)
```

No NetworkX subgraph materialization for blast radius hot path. NetworkX used only for: `shortest_path`, `god_nodes`, community detection.

## Pipeline

```python
# src/loom/ingest/pipeline.py
async def index_repo(
    repo_path: Path,
    graph: LoomGraph,
    *,
    docs_path: Path | None = None,
) -> IndexResult:
    files = walk_repo(repo_path)
    existing = await graph.get_content_hashes()
    changed = [f for f in files if sha256_of(f) != existing.get(str(f))]

    with ProcessPoolExecutor(max_workers=min(cpu_count(), 8)) as pool:
        parsed = list(pool.map(_parse_file, changed, chunksize=20))

    nodes, edges = resolve_calls(parsed)

    for path, fn, fe in group_by_path(nodes, edges):
        await graph.replace_file(path, fn, fe)

    await compute_communities(graph)
    await compute_coupling(graph, repo_path)
    await mark_dead_code(graph)

    return IndexResult(...)
```

**No embedding pass. No linker pass. No drift pass. No Jira fetch.**

Cross-file call resolution is a serial pass because it needs the global symbol map from all parsed files.

## Incremental sync

```python
# src/loom/ingest/incremental.py
async def sync_paths(
    graph: LoomGraph,
    repo_path: Path,
    *,
    old_sha: str | None = None,
    new_sha: str | None = None,
) -> SyncResult:
    if old_sha and new_sha:
        changed = git_diff_files(repo_path, old_sha, new_sha)
    else:
        changed = [p for p in walk_repo(repo_path)
                   if sha256_of(p) != await graph.get_file_hash(str(p))]

    with ProcessPoolExecutor(max_workers=min(cpu_count(), 8)) as pool:
        parsed = list(pool.map(_parse_file, changed, chunksize=20))

    nodes, edges = resolve_calls(parsed)
    for path, fn, fe in group_by_path(nodes, edges):
        await graph.replace_file(path, fn, fe)

    if len(changed) / max(len(walk_repo(repo_path)), 1) > 0.05:
        await compute_communities(graph)

    return SyncResult(...)
```

### `loom install` behavior

Detects presence of and writes MCP config entry to:
- `~/.claude/mcp.json` (Claude Code)
- `~/.cursor/mcp.json` (Cursor)
- `~/.codeium/windsurf/mcp_config.json` (Windsurf)
- `~/.config/zed/settings.json` (Zed)
- `~/.codex/mcp.json` (Codex)

Writes entry:
```json
{"loom": {"command": "loom", "args": ["serve"]}}
```

Writes `.git/hooks/post-commit`:
```sh
#!/bin/sh
loom sync \
  --old-sha "$(git rev-parse HEAD~1 2>/dev/null || git rev-parse HEAD)" \
  --new-sha "$(git rev-parse HEAD)"
```

`chmod +x .git/hooks/post-commit`. Prints detected platforms and what was written.

Optional: ships `hooks/session-start.sh` that prints graph-first guidance. User symlinks/sources from their own Claude Code `SessionStart` hook if desired.

## Conventions

- Python 3.12. `ruff` for linting. `mypy --strict` must pass.
- Pydantic v2 throughout. No raw dict access.
- Async at CLI entry points. `asyncio.to_thread` for sqlite3 calls (sync).
- `ProcessPoolExecutor` for CPU-bound parsing. SQLite writes remain serial (single-writer).
- No silent fallbacks — raise or log WARNING minimum.
- Docstrings: Args/Returns on public functions only.
- Comments only where WHY is non-obvious. Never WHAT.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| `tree-sitter-language-pack` API differs from per-language pkgs → parser tests fail | Day 6 isolates this swap. Rollback: re-add individual deps if blocker. |
| SQLite recursive CTE slow on 100k+ node graphs | Test on large repo (e.g. cpython). If slow, fall back to NetworkX load (kept as secondary path). |
| FTS5 absent on some Python sqlite3 builds | Detect at startup. Fall back to `name LIKE ?` with warning. |
| ProcessPoolExecutor pickle failure on tree-sitter Tree objects | Parse to plain dicts/dataclasses before returning from worker. |
| Rewriting 34 tests is a long tail | Budget day 7 for it. If tests block > day 8, ship with partial coverage and file follow-up tickets. |
| Deleting ticket/drift breaks downstream users | None yet (pre-v0.1 surface). Document in CHANGELOG. |
